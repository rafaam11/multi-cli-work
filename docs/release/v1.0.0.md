# Multi CLI Work v1.0.0

## Summary

First installable release. Multi CLI Work is a local Windows workspace that runs Codex, Claude Code, and PowerShell sessions grouped by project, sharing project identity and metadata with Harness Manager through `~/.harness-manager/projects.json`.

## Features

- Project tree with multiple nested sessions per project and a single focused terminal viewport.
- Projects discovered from local Claude and Codex history, plus manually added folders. Display name, status, memo, and hidden flag are editable and shared with Harness Manager; hidden projects can be revealed and relinked.
- Unified session states (`starting`, `working`, `awaiting-input`, `awaiting-approval`, `idle`, `exited`, `error`). Windows notifications fire only when an unfocused session starts waiting for input or approval, and repeat states are suppressed.
- Closing the window hides the app to the tray and keeps PTYs alive. Explicit Quit confirms, then stops them. After a restart, projects, tabs, and bounded scrollback are restored; AI sessions resume through the visible Resume action.
- Shared registry is crash-safe: cross-process locking, atomic replacement, validated backup, and restore-from-backup when the primary file is corrupted.
- Automatic updates from GitHub Releases, with a sidebar badge and a tray entry for manual checks.

## Compatibility

- Harness Manager keeps its own board data; only project identity and project metadata live in the shared registry. Both apps may run at once.
- Multi CLI Work works with no Harness Manager installed: a missing registry file is created and owned by whichever app writes first.
- Registry contract: [`docs/superpowers/specs/registry-contract.md`](../superpowers/specs/registry-contract.md).

## Install

`Multi-CLI-Work-Setup-1.0.0.exe` — per-user NSIS installer, no administrator rights. Not code-signed, so SmartScreen requires **More info → Run anyway** on first run.
